Why Memova:
Foundations

解释当前产品背后的稳定原则：Memova 为什么必须 context-first、human-led、local-first，并且比它所整理的信息更平静。

Chapter 03 · Page 01Updated 2026-07-194 min read

本页是 FOUNDATION。若与 2026-07-19 的当前产品定义冲突，Chapter 02 优先。

30 Seconds

Agent Quick Read

Memova 采用 context-first 的个人 AI 路线：用户继续自然地说、写、开会与工作，Memova 负责积累可用 Context。个人记忆默认私密、用户拥有、可导出且可由授权 Agent 读取；系统可以主动整理、建议和准备，但分享与高影响行动必须由人确认。Page → Book → Library 为这些原则增加了可见、可检查、可积累的产品界面，它延伸而不替代 Notes → Memory → Actions。

Principle 01
STABLE PRINCIPLE

从 Tool AI 到 Ambient AI

人不应该为了使用 AI，先成为 Prompt 工程师、工作流运营者或多个 Agent 的管理员。

真实的 Context 已经存在于笔记、语音、会议、手写、决定、关系与持续项目中。Memova 应进入这些自然活动，把它们转化为结构化记忆，再安静地准备下一步。

Tool-first：打开工具 → 重建 Context → Prompt → 搬运结果
Context-first：自然生活与工作 → Context 累积 → Memova 准备重要结果

Ambient 不等于不可见或完全自治。它意味着低摩擦捕捉、主动协助，以及清晰可见的检查与控制时刻。

Principle 02
STABLE PRINCIPLE

Personal Superalignment

一个真正有用的个人 Agent 必须对齐某个具体的人，而不只是一个平均化的“用户”。

理解 Context

保留经历、人物、项目与决定之间的连续关系，而不是只读取当前 Prompt。

理解 Intent

持续理解目标、偏好、优先级和约束，并允许它们随时间改变。

接受 Correction

用户的修改、反馈与反例应更新系统对这个人的理解。

保持 Inspectable

记忆与来源应能被用户查看、纠正、移动、撤回和导出。

INTERPRETATION 个人图书馆是对齐状态的一种可读形式：人可以检查系统记住了什么，授权 Agent 也不必从孤立 Prompt 中重新猜测这个人。

Principles 03–05

让能力增长，同时让控制保持清晰

HUMAN-LED

人仍然是意图和高影响行动的最终权威

Memova 可以自动捕捉、关联、识别风险、建议、排序和准备；发送、发布、安排、提交或改变重要外部状态前必须确认。连接账号不等于同意发布。

LOCAL-FIRST

记忆默认私密、用户拥有、可导出、Agent-readable

Local-first 是信任与连续性属性，不是所有计算都必须发生在一台设备上的承诺。用户应知道持久记忆在哪里，能迁移它、授权有限访问并撤销权限。

LOW COGNITIVE LOAD

Memova 应比它整理的信息更平静

复杂度可以存在于系统内部，但只应在用户需要纠正、批准、移动或检查时出现。默认体验不要求维护复杂目录、密集 Dashboard 或 Agent 路由。

Useful Outcomes
STABLE PRINCIPLE

记忆必须改善理解与行动

Memova 的价值不在于存储更多信息，而在于让理解、判断、沟通和行动变得更好。

原有的 Notes → Memory → Actions 仍然成立。当前定义在记忆与行动之间增加了一个可见结果：一张用户愿意保留或分享的 Page。它可以保持私密、支持与同事的沟通、成为社媒发行素材，或为 Agent Hands 提供 Context。

CURRENT DEFINITION
公开发布是可选动作。只把一个私密 HTML 书页分享给同事、朋友或家人，也已经是完整的产品结果。
Extension
CURRENT DEFINITION

为什么自然走向 Page → Book → Library

Complete Context个人经历、项目、人物、目标与约束
Page可检查、编辑、保留或分享的记忆单元
Book相关书页组成持续演化的主题叙事
Library人和授权 Agent 共用的长期 Context
Better Next Page过去的记忆提高下一次理解与输出

这不是替换。它为 context-first AI 提供可检查的记忆对象，为 human-led sharing 提供确认边界，为 local-first memory 分离权威源与外部投影，也让用户不必维护文件、Agent 和工作流。

INTERPRETATION “个人出版社”让输出容易理解，但发布不是 Memova 的全部。真正的长期资产是不断演化的私密 Context；它让每张书页比通用内容生成器更对齐这个人。

Guardrails

这些原则排除了什么

  • Memova 不是以多平台排程为核心的 Social Scheduler。
  • 不要求用户管理一组 Agent，也不静默执行高影响动作。
  • 不把 HTML、图片或社媒 Post 当作唯一知识源。
  • 不强迫用户发布到公域才能获得价值。
  • 不把私人记忆封闭在不可导出的黑箱 Cloud Profile 中。
