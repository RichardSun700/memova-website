The Memova
Product System

把稳定的 Context + Action 架构与新的 Page → Book → Library 产品表面放进同一张系统图。

Chapter 03 · Page 02Updated 2026-07-195 min read

Page 是新增的可见价值单元；Inbox、Knowledge Base 和 Agent Hands 没有被替换。

30 Seconds

Agent Quick Read

Memova 仍然遵循 Human ↔ Inbox & Brief ↔ Knowledge Base ↔ Agent Hands → Outputs。Inbox & Brief 捕捉自然交互并组织成对齐的 Context；Knowledge Base 保存结构化、用户拥有的记忆；Agent Hands 使用这些 Context 准备或执行经批准的工作；Outputs 把结果交还给人。Page → Book → Library 增加一个可见循环：结构化 Context 成为 Private Page，相关 Page 组成 Book 与 Library，经过用户确认的快照再成为 Shared Page 和平台发行素材。

Stable System
FOUNDATION

Context 把自然输入连接到有用结果

Human保持自然地思考、说话、书写、开会和决定
Inbox & Brief捕捉输入，组织目标、约束与执行意图
Knowledge Base持久保存人、项目、决定、承诺与来源
Agent Hands选择合适工具，准备或执行已授权的工作
Outputs把可读、可分享或可执行的结果交还给人

用户通过自然活动进入系统，而不是通过 Agent 列表；Memory 是 Capture 与 Outcome 之间的连接层，而不是流程末端的 Archive。

Four Layers

四个稳定产品层

01 · ENTRY

Inbox & Brief

Inbox 接收快速想法、文字、语音、会议、图片、白板、手写、已有视频与项目材料。Brief 把原始输入组织成相关人物、项目、目标、约束、未决问题与期望结果。

创业者约三分钟语音是当前最清晰的第一入口，但不是 Entry Layer 的永久边界。

02 · MEMORY

Knowledge Base

保存人物和关系、项目状态、决定与原因、承诺与负责人、事件、偏好、重复模式，以及来源、纠正和权威状态。

应当 local-first、可导出、用户拥有，并可由获得授权的 Agent 阅读。

03 · EXECUTION

Agent Hands

使用 Memova Context 准备日历、跟进消息、任务、研究、代码、Brief 或发行包。内部可以连接多个工具，用户不需要负责路由。

默认顺序：观察和结构化 → 建议和准备 → 展示后果 → 请求确认 → 行动。

04 · VALUE

Outputs

原有输出包括计划、草稿、日历、任务、提醒、报告与 Agent 完成的工作。当前定义新增 HTML、长图、多图、平台适配内容与用户视频包装。

输出是派生物；修改平台文案不能静默改写 Private Page。

Current Extension
CURRENT DEFINITION

Page → Book → Library 是记忆循环，不是第五个孤立模块

Structured ContextInbox 与 Knowledge Base 提供完整背景
Private Page可编辑、可检查的权威私人记忆对象
Book按项目、主题、人物与时间形成演化叙事
Library人类阅读界面与授权 Agent Context Surface
Future Context过去书页提高下一次 Brief、Page 与 Action
Knowledge Base → Private Page → Book → Library → future Context
Private Page → 用户检查与确认 → Shared Page → Distribution Assets
Knowledge Base ↔ Agent Hands → operational Outputs

这个组合增加三个能力：Page 是可见的第一价值单元；Book 与 Library 让积累自然出现；Shared Page 在私人记忆与外部输出之间建立可审查的确认边界。

Object Contract

每个对象只承担一种责任

PRIVATE · CANONICAL

Private Page

由原始输入与既有 Context 形成的持久记忆对象；可含来源、推断、敏感信息、纠正与行动可能。默认私密，属于 Knowledge Base。

SHARED · SNAPSHOT

Shared Page

经过用户确认的投影。MVP 中一张 Private Page 最多一个分享快照；私人源更新不会静默改变外部页面。

DERIVED · PLATFORM

Distribution Assets

平台特定派生物；可以改变长度、裁剪、顺序和文案，但不会成为新的知识源。

RELATED · EVOLVING

Book

相关 Private Pages 形成的持续叙事，不是静态 Folder。Memova 建议归书，用户可确认、改名、移动、合并或拆分。

OWNED · RETRIEVABLE

Library

Book 与关系的用户空间；同时服务人类阅读和 Agent Context，必须保留来源、状态与权限。

ACTION · AUTHORIZED

Operational Output

由 Agent Hands 准备的任务、日历、消息、代码或报告；依风险匹配确认，不与分享素材混为同一对象。

Continuity

这个具体方向 改变什么，又保留什么

CHANGED

新增可见价值与积累模型

  • Page 成为第一个用户可见价值单元。
  • Memory 可先生成完整作品，再触发外部 Action。
  • 私域分享是完整结果，公域发行可选。
  • Book 让长期积累无需人工维护 Folder。
  • Sharing 拥有明确的 Snapshot Boundary。
UNCHANGED

Context 与人类控制仍是基础

  • 自然交互仍是入口。
  • Knowledge Base 仍为人和 Agent 提供 Context。
  • Agent Hands 仍是执行伙伴，不是主界面。
  • 重要动作仍然 Human-led。
  • 私人记忆仍与外部 Output 分离。
  • Useful Action 仍是长期承诺的一部分。
Invariants

实现时不能丢失的七条规则

  1. Raw Input、Canonical Memory、Shared Projection 与 Platform Derivative 拥有不同身份。
  2. 每个外部投影都能追溯到 Private Page 与确认事件。
  3. 发布失败不能污染或阻塞 Private Page 与 Shared Page。
  4. Book 组织 Canonical Pages，而不只是它们的公开版本。
  5. Library 改善未来 Context，但不静默改变已经分享的快照。
  6. Agent 只获得其任务所需的 Context 与权限。
  7. 用户能检查、纠正、移动、撤回与导出自己的记忆。
