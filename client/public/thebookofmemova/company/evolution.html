到会生长的产品书

一条带权威状态的演进线：哪些原则延续至今，哪些研究支持了方向，哪些探索没有成为当前承诺。

Chapter 04 · Page 077 min readUpdated 2026-07-19

时间线解释当前产品从哪里来，但较早概念不能覆盖当前产品书。

30 Seconds

Agent Quick Read

Memova 没有从“笔记产品”突然变成“发布工具”。五月形成的基础是：人通过自然输入进入 Inbox & Brief，Knowledge Base 组织人物、项目、决策和承诺，Agent Hands 准备行动，人类保持最终控制。随后，HTML artifact 研究说明 agent 输出需要稳定、私密、可引用的页面；用户元认知研究说明 agent 还必须标记意图、证据、置信度和行动边界。七月的当前定义把这些线索组合成 Context → Private Page → Shared Page → Book → Library。硬件仍是历史探索。

Authority

时间线不是投票

越早出现的想法不因“历史更久”而拥有更高权威。当前定义优先，基础负责解释，证据支持判断，历史只保留 provenance。

CURRENT > FOUNDATION > EVIDENCE > HISTORICAL > OPEN

旧文档中没有被当前产品书重申的指标、功能范围、硬件规格或市场说法，不应被视为当前事实。

May → July

六个阶段，汇合成一个产品循环

MAY 14
EVIDENCE

从 AI Notebook 市场看到 Context 缺口

会议与笔记只有在进入持续 Context、支持后续决策和行动时才产生长期价值。研究提供方向证据，不证明用户需求已被验证。

MAY 18
HISTORICAL

把手写与音频想成环境化入口

Memova Pen 探索强化了自然输入、可见隐私和本地控制，但具体形态、参数与原型阶段没有进入当前硬件路线。

MAY 20
FOUNDATION

形成端到端产品层

Human → Inbox & Brief → Knowledge Base → Agent Hands → Outputs → Human confirmation，确立了 Context 与协调层的公司骨架。

MAY 26
EVIDENCE

HTML 从临时输出变成记忆对象

Agent 产出的报告、brief 和页面需要稳定、私密、可浏览的链接；Memova 的机会是继续保存来源、关系、决策与下一步。

JUN 30
FOUNDATION

从记住用户走向校准理解

Agent 需要说明意图假设、证据、置信度和行动边界，并从用户的接受、修改和纠正中学习。

JUL 19
CURRENT

Page 成为核心产品对象

创业者的语音与文字生成私密书页；用户可确认分享或发行；页面自动归书并成为下一张页面的 Context。

Continuity

没有改变的公司骨架

01

Context-first

从人的真实互动中理解 Context，而不是每次要求用户重新解释。

02

Human-led

系统主动整理和准备；重要发送、公开和承诺由用户确认。

03

Actionable memory

记忆不仅回答发生了什么，也帮助判断接下来做什么。

04

People & projects

人物、项目、决策与承诺比孤立文件更接近真实生活结构。

05

Local-first & portable

长期记忆应可导出、可删除、对 agent 可读，并减少平台锁定。

06

Low cognitive load

用户看到平静、可理解的下一步，而不是需要维护的 AI 控制台。

Convergence

当前定义把抽象架构变成可感知对象

过去较抽象的表达当前可感知的产品对象
Outputs一张愿意保存或分享的 Page
Knowledge Base自动形成的 Books 与 Personal Library
Agent handoff带来源、状态和关系的 HTML 书页
Action loop分享、发行、反馈与下一张书页的闭环
广泛候选用户创业者优先，学生与 SMB 后续
多种可能入口语音和文字优先，其他媒体补充
Boundaries

哪些探索没有成为 CURRENT

  • HISTORICAL 以硬件为前提的产品路线。
  • HISTORICAL 历史研究中的市场数字、benchmark 门槛与工业设计参数。
  • LATER 九个平台全部直发、复杂团队审批和完整社交媒体日历。
  • LATER AI 生成新的动态视频画面或自动社交互动。
Evidence Gaps

仍需真实使用回答

  • 创业者是否愿意从一段短语音生成并保留第一张书页？
  • 私域分享是否比公域发布更早、更频繁地产生价值？
  • 自动归书是否让用户感到图书馆正在形成，而不是增加整理负担？
  • 人工确认能否在隐私安全与低摩擦之间取得平衡？
  • 一个平台的真实发行能否成为增益，而不是分散核心体验？

这些是验证问题，不能被写成已经证明的结果。
