当前输入和原始证据优先于旧记忆。HTML 是记忆的阅读投影，不是唯一知识源。

30 Seconds

Agent Quick Read

Memova 的核心接口是用户已经会做的事：说话、写字、开会、拍照和记录想法。系统把这些输入连接到人物、项目、决策、承诺和历史 Context，同时保留来源、置信度与行动边界。当前输入和原始证据始终优先于旧记忆；系统保存可检查的判断字段，而不是隐藏的思维过程。HTML 是可分享投影。Memova Pen 是一种可能的手写与音频入口，但仍属历史探索，不在当前 MVP 内。

Capture

自然接口，而不是 AI 控制台

用户在表达和工作，Memova 在后台把活动变成可阅读、可确认和可积累的记忆。

CURRENT · 01

语音与会议

捕捉复盘、观点、决策、人物关系和下一步线索，是创业者 MVP 的主要入口。

CURRENT · 02

文字与快速笔记

补充事实、纠正推断、固定措辞，并明确哪些内容可以进入分享快照。

SUPPORTING · 03

图片、纸张与白板

作为证据和 Context 进入页面；识别结果必须保留与原图的关系，不能把推断伪装成原文。

LATER INPUT · 04

手写与已有媒体

手写可结合时序和音频；已有视频可剪裁、加字幕和封面，但当前不生成新的动态画面。

Memory Model

记忆不是一堆转录

原始笔记、转录和 OCR 是证据层；长期记忆需要能被人和 agent 稳定引用的对象。

对象需要保留什么帮助回答什么
People身份、关系历史、偏好、承诺、沟通语境这件事关于谁？怎样跟进才合适？
Projects目标、状态、约束、阻碍、相关页面它属于哪一段持续工作？
Decisions决定、理由、证据、变化时间为什么这样做？现在是否仍有效？
Commitments谁答应了什么、时间、状态、依赖下一步是什么？谁需要确认？
Pages & Books叙事、来源、关系、版本和权威状态如何保存、分享并再次使用？
Understanding

Agent 要知道自己对用户理解得有多准

普通记忆回答“用户说过什么”。Memova 还要说明系统推测用户想完成什么、依据是什么，以及何时应该确认。

Current Input用户刚刚说、写或确认的证据
Intent明确请求与一到多个意图假设
Calibration证据、置信度与猜错代价
Response Mode回答、询问、起草、规划、提醒或等待
Feedback接受、修改、忽略、删除或纠正

这些字段应以结构化、可审查的数据保存。Memova 不需要保存或展示模型的隐藏推理过程；需要的是结论、证据、置信度和行动边界。

系统可以多问一次，但不能从低置信度推断直接跨越发送、公开、排期或承诺的边界。
Projection

HTML 是书页投影，不是记忆源头

同一段记忆需要同时服务用户本人、被授权的读者和未来 agent，但三者看到的表面不必相同。

Raw Evidence语音、文字、图片、文件与历史 Context
Structured Memory人物、项目、决策、承诺、来源与状态
Canonical Page完整私密页面与用户确认
HTML Page面向人的分享快照
Distribution长图、多图、视频与平台版本

分享 HTML 与社交素材可以改变长度、顺序和视觉形式，但不能反向覆盖私密书页，也不能成为事实的唯一存档。

Historical
HISTORICAL · EXPLORATION

Memova Pen 是接口研究，不是当前承诺

早期研究设想过把手写轨迹、音频和 Context 带入 Memova 的书写设备，并探索“记忆脊柱”等工业设计语言。

值得保留的是四个接口洞见：手写与音频的时间关系可能比单独 OCR 更有语境；录音和同步状态必须清楚可见；本地处理、导出与删除是信任的一部分；硬件如果存在，应让人更自然地捕捉 Context，而不能成为使用 Memova 的门槛。

  • 没有硬件立项、量产或上市承诺。
  • 具体形态、材料、规格和原型阶段不属于当前产品事实。
  • 先验证软件中的语音、文字、私密页面和分享闭环。
Quality

接口质量检查

  • 用户能分辨原始证据、系统推断和已确认事实。
  • 用户能知道内容是私密、待分享还是已形成分享快照。
  • 旧记忆发生冲突时不会覆盖当前证据。
  • Agent 可通过 Page ID、来源、关系和状态安全重用内容。
  • 高影响动作清楚显示确认边界。
  • 录音、上传、同步和公开状态不会为了“智能感”被隐藏。
