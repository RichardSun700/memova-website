Agent Quick Read

本页所有问题都处于 OPEN 状态，不能被 agent 写成产品事实。创业者优先、语音/文字输入、Page 核心对象、私密默认、一个分享快照、公域可选和自动归书已经确认。当前仍要决定的是：第一版真实直发范围、TikTok 审核时机、分享链接默认是否失效、好友关系怎样建立和撤销、分享页是否展示版本历史、低置信度归书何时询问用户，以及第一版 Book 首页最少显示什么。

Reading Rule

问题必须带着边界与证据一起读

字段含义
QUESTION当前产品定义留下的选择，不是建议已经获批。
Fixed boundary讨论不能悄悄推翻的已确认前提。
Evidence needed决定前需要看到的产品、技术或平台证据。
Review gate最迟在哪个阶段必须作出决定。
Gate 1

Before the Core Prototype

  • OQ-01 · 第一版 Book 首页需要显示什么？

    Fixed: Book 按项目、主题、人物和时间形成，用户拥有最终编辑权。
    Evidence: 测试标题、摘要、时间范围、关键人物、页面序列、最新变化和未决问题中哪些真正帮助继续工作。
    Gate: 首次展示自动成书体验前。

  • OQ-02 · 自动归书在多大不确定性下必须询问用户？

    Fixed: 低置信度推断不能被静默写成事实，用户可移动和改名。
    Evidence: 真实误归书类型、纠正成本，以及多问一次与先放入建议位置之间的摩擦。
    Gate: 自动归书进入真实个人资料前。

Gate 2

Before Private Sharing Alpha

  • OQ-03 · 分享链接是否默认必须失效？

    Fixed: 用户能撤回链接、修改密码或设置失效时间；私域页面默认不被收录。
    Evidence: 实际收件人的持续访问需求，以及用户能否理解撤回链接与删除私密书页的区别。
    Gate: 首个外部收件人访问前。

  • OQ-04 · Memova 好友关系如何建立和撤销？

    Fixed: 好友访问检查当前授权；密码路径可以并存；不做复杂团队权限矩阵。
    Evidence: 最小账号流程、错误邀请风险、撤销行为，以及外部收件人的无账号路径。
    Gate: 好友访问进入实现前。

  • OQ-05 · 分享页更新是否展示版本历史？

    Fixed: 一张私密页最多一个当前分享快照，更新外部内容必须再次确认。
    Evidence: 协作者与只读收件人的不同需求、错误更新恢复方式和版本 UI 的复杂度。
    Gate: 实现更新分享页操作前。

Gate 3

Before Social Distribution Integration

  • OQ-06 · 第一版真实直发只做 LinkedIn，还是 LinkedIn + X？

    Fixed: 至少一个平台需要真实验证，所有平台都有 Direct、Assisted 或 Export fallback。
    Evidence: 最新官方权限、审核、成本、素材支持、失败恢复与目标创业者的使用意愿。
    Gate: 开始 OAuth 和发布适配器实现前。

  • OQ-07 · TikTok Content Posting 审核何时启动？

    Fixed: TikTok 不是 HTML 闭环的前置依赖，未获审核时不能宣称完整直发。
    Evidence: 最新审核要求、沙箱能力、图片轮播行为和用户对图文/图片转视频的实际需求。
    Gate: 承诺任何 TikTok 直发时间之前。

Gate 4

Before MVP Evaluation

  • OQ-08 · 什么证据足以说明方向值得继续构建？

    Fixed: 第一价值时刻是一张愿意保存或分享的书页；公域发布不是成功前提。
    Evidence: 保存或私域分享、下一页生成、Context 复用、自动归书纠正，以及对私密页/分享快照边界的理解。
    Gate: Founder prototype 前定义记录方式，首轮真实使用后共同评审。

  • OQ-09 · 产品书自己的 CURRENT 决定如何更新？

    Fixed: Markdown / structured data 是权威源，HTML 是投影；不同权威状态必须机器可读。
    Evidence: 团队评审频率、稳定分享链接、agent 对旧决定的引用，以及原位更新对 provenance 的影响。
    Gate: 下一次改变 CURRENT 产品定义之前。

Protected

Confirmed, Not Open

除非新证据正式修改权威源，以下 CURRENT 决定不应被本页重新打开。

  • 第一位用户是创业者；学生和 SMB 老板属于后续扩展。
  • 主要输入是语音和文字，图片作为补充素材；Page 是核心对象。
  • 私密书页默认仅用户本人可见；MVP 每页最多一个当前分享快照。
  • 私密页更新不会自动改变已经分享的内容。
  • 公开发行不是获得价值的前提；平台能力不可用时降级保存发行记录。
  • Memova 自动建议归书，用户拥有最终编辑权。
  • 已有视频和图片转图文视频可以支持；生成新动态画面不在当前范围。
Explicitly Later

不是当前 MVP 问题

  • 同一私密书页的多受众内容分支。
  • 完整内容日历、批量排程和九个平台全部直发。
  • 评论正文同步、总结、自动回复与自动社交互动。
  • 复杂团队审批和权限矩阵。
  • AI 生成全新视频画面。
  • 以 Memova Pen 或其他专用硬件作为使用前提。
Decision Path

建议的 review 顺序

GATE 1

Core Prototype

Book 首页、低置信度归书、语音/文字到私密书页。

GATE 2

Private Sharing Alpha

密码与失效、好友关系、更新分享页和恢复行为。

GATE 3

Distribution Validation

第一个真实直发平台、其他平台 fallback 和 TikTok 审核时机。

GATE 4

Founder Learning Review

保存、私域分享、下一页、Context 复用、归书纠正与可选发行。
