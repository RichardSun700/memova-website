CHAPTER 02

First User, First Value

第一位用户不是社媒运营者，而是每天产生大量高价值 context、却没有时间把它整理为成果的创业者。

Founder · MVP

创始人表达

会后复盘、产品判断、行业观察、项目进展和尚未整理的观点。

Student · Next

学习成果

课堂、论文、读书和项目变成学习书页与作品集。

SMB · Next

经营知识

产品知识、客户故事和经营经验变成案例与教育型内容。

MVP 的成功时刻：用户第一次从约三分钟语音得到一张愿意保存或分享给某个人的书页。
CHAPTER 03

The Minimal Product Model

MVP 只有四个用户能理解的对象。复杂的多受众版本以后再做。

01 · Source

私密书页

完整语音、资料、项目 context、扩展笔记、来源和敏感信息。

02 · Snapshot

分享书页

每页最多一个分享快照；支持好友、密码或公开访问。

03 · Formats

发行素材

HTML、长图、多图、LinkedIn、TikTok、X 和图文视频。

04 · Memory

书

按项目、主题、人物和时间自动归书，用户掌握最终编辑权。

Source-of-truth rule私密书页是知识源；分享页是确认后的快照；平台素材只是发行格式，不能反向污染知识源。
CHAPTER 04

Core User Flow

1–3

Capture → Understand → Page

用户说约三分钟，补充文字或媒体。Memova 调用项目历史，识别人物、事实、决策和敏感信息，生成私密书页。

4–6

Choose → Review → Access

用户选择只保存、私域分享或公域发行；确认分享内容后，设置好友、密码、失效时间或公开。

7

Optional Social Distribution

逐个平台预览账号、文案、素材顺序和声明，再分别确认。任何平台失败都不影响 HTML。

8–9

Auto-book → Feedback

私密书页自动归书。永远保存发行记录；权限允许时再异步同步基础指标。

PRIVATE VALUE

不发公域也成立

  1. 只保存到私人图书馆
  2. 好友登录查看
  3. 外部收件人输入密码
OPTIONAL REACH

需要时再发行

  1. 公开 HTML
  2. LinkedIn 知识内容
  3. TikTok 图文与 X Thread
CHAPTER 05

Privacy and Version Rules

完整 context 是产品优势，也是最高风险。分享必须是一次明确的人类决策。

  • 私密书页默认仅用户本人可见。
  • 每张私密书页在 MVP 中最多一个分享快照。
  • 私密页变化不会静默更新外部 HTML。
  • 分享页可由 Memova 好友登录访问，也可由持有密码的访客访问。
  • 密码不以明文保存；链接可撤回、改密或设置失效。
  • 私域页面默认 noindex；媒体资源不能绕过页面权限。
  • 一页多受众版本明确放入后续路线，不进入 MVP。
CHAPTER 06

Distribution Strategy

Priority 01

LinkedIn

最符合创业者知识型内容。支持文字、视频、多图和 PDF 文档。

Priority 02

TikTok

支持视频和图片轮播；Direct Post 需要平台审核并遵守显式发布选项。

Priority 03

X

文字、图片和视频发布可行，但 API 按量付费，应进入套餐成本。

每个平台都必须优雅降级

模式用户体验适用情况
Direct PublishMemova 内确认后直接发布官方接口可用且应用已获批准
Assisted Publish素材准备完成，打开平台由用户完成最后确认审核中或平台功能必须原生完成
Export Only下载图片、视频并复制平台文案接口不可用或用户未连接账号
Video boundary可以处理用户已有视频，或把图片做成图文视频；不直接生成新的动态画面或视频帧。
CHAPTER 07

Feedback Without Blocking

数据权限是增强项，不是发布前提。

ALWAYS · A

发行记录

平台、账号、状态、时间、Post ID、链接、错误和重试状态。

WHEN ALLOWED · B

基础指标

浏览或曝光、点赞、评论数、转发数与最近同步时间。

X 的公开指标相对直接；TikTok 授权视频可读取基础指标但图片轮播需验证；LinkedIn 指标需要额外 Community Management 权限。拿不到时退回 A，不能把“无权限”显示成“零表现”。

CHAPTER 08

MVP Scope

MUST HAVE

第一版必须完成

  • 语音/文字 → 私密书页
  • 单一分享快照
  • 好友和密码访问
  • HTML、长图和多图
  • 三平台适配预览
  • 至少一个真实直接发布
  • 其他平台 fallback
  • 自动归书建议
  • 发行记录与可选指标
NOT NOW

明确延后

  • 多受众分享版本
  • 完整社媒日历
  • 九平台全部直发
  • 评论正文和自动回复
  • 复杂团队审批
  • 生成式视频画面
  • 以分析仪表盘为主的首页
CHAPTER 09

Competitive Position

iOS 多平台发布已经被验证。它是基础设施，不是 Memova 的护城河。

产品值得学习Memova 的差异
Hookle移动优先、多平台同时发布、多图从完整 context 生成值得保存的 Page；分享与社媒只是 Page 的出口；Page 会自动归书并构成长期图书馆。
Crosspost逐平台预览、统一发布、Master Profile
Socialync轻量一键发布、平台专属 caption、视频 transcript
Buffer / Publer / Later成熟平台接入、排程、内容库与分析不以 Calendar 为核心，不要求用户先成为内容运营者。
Existing tools: Asset → Post → Calendar → Platform Memova: Context → Page → Book → Library └→ Private share / Social distribution
竞品是多平台邮局。Memova 应成为个人出版社和私人图书馆。
CHAPTER 10

Success Criteria

Activation

第一次价值

完成三分钟语音；得到愿意保存或分享的书页；完成一次私域分享或平台发布。

Quality

可信与可控

无未经确认的敏感信息；用户清楚私密与分享边界；移动端无裁切。

Retention

第二张书页

一周内再次生成；能复用项目历史；开始看到一本书自然形成。

Distribution

真实分享

愿意把 HTML 发给同事、朋友、家人或投资人，或把发行版本发布到平台。
