# Memova Company Memory Book

> **Direct Link · Noindex**
> 这不是一份普通公司文档，而是 Memova 生成的关于 Memova 自己的第一本记忆之书。

## 两句话理解这本书

> **人的价值，不在于被记录，而在于持续被理解。**

> **Memova 的第一把尖刀，是把创业者不断产生的 Context 变成愿意保存、分享并最终归入个人知识之书的 Page。**

第一句回答 Memova 为什么存在；第二句回答现在从哪里切入。它们不是两个方向：Chapter 01 给出长期用户承诺，Chapter 02 把承诺变成一个具体、高频、有痛点的产品入口。

## 阅读顺序

### Chapter 01 · 用户宣言——被记住的未来

**状态：MANIFESTO**

先理解 Memova 对用户的承诺：记忆不是静态数据，而是持续发生的理解与连接；用户应拥有、纠正并持续对齐自己的记忆。

### Chapter 02 · 第一把尖刀

**状态：CURRENT · 产品决策最高权威**

用十分钟阅读三页：

1. **[第一把尖刀](memova-now.md)** — 创业者切口、第一次价值时刻，以及为什么这是具体方向而不是公司转向。
2. **[The Product Model](product-model.md)** — Private Page、Shared Page、Distribution Assets 与 Book。
3. **[Experience & Sharing](experience-sharing.md)** — 输入、私域分享、公域发行、隐私、归书和 MVP。

### Chapter 03 · Why Memova

**状态：FOUNDATION**

解释当前产品为何成立：Ambient AI、个人超级对齐、Human-led Agency、Local-first Memory、低认知负担，以及输入—记忆—行动系统。

### Chapter 04 · How We Got Here

**状态：EVIDENCE / HISTORICAL / OPEN**

保存产品演进、证据、历史探索和未决问题，让团队与 Agent 能追溯为什么形成今天的判断，又不会把旧探索误读为当前承诺。

## 阅读权威

```text
CURRENT > MANIFESTO > FOUNDATION > EVIDENCE > HISTORICAL > OPEN
```

- **CURRENT**：已确认的当前产品定义，决定现在做什么。
- **MANIFESTO**：稳定的用户价值与长期承诺，解释为何存在，但不静默扩展 MVP。
- **FOUNDATION**：稳定公司原则，解释当前定义。
- **EVIDENCE**：市场、研究、产品或实现证据。
- **HISTORICAL**：为追溯而保留的旧探索。
- **OPEN**：尚未决定，不得写成产品事实。

## 这本书本身就是产品 Demo

```text
公司语音、会议、决策、研究与项目 Context
                    ↓
有来源、状态、摘要和关系的 Page
                    ↓
相关 Page 组成 Chapter，并汇入 Book
                    ↓
可供人和 Agent 快速阅读的 Memory Library
```

Markdown 是可追溯的记忆源，HTML 是可阅读、可分享的投影；分享与发行不会覆盖私人知识源。用户不需要先设计复杂文件夹，只要继续表达、工作和思考，Page 就能持续形成 Book 与 Library。
