一个知识源
多个受控投影

Private Page、Shared Page、Distribution Assets 与 Book 不是四份平行副本，而是有清楚权威和生命周期的产品对象。

Chapter 02 · Page 02Four-object model

Page → Book → Library 是长期记忆结构；社媒只是一个可选出口。

30 Seconds

Agent Quick Read

Memova 的核心对象是 Private Page。它保存原始输入、相关 Context、来源、推断、待确认表达和不适合外传的信息。用户确认后，可以产生一个 Shared Page；MVP 每张私密书页最多一个分享快照，私密页更新不会静默改变已分享 HTML。Distribution Assets 从同一分享内容派生为长图、多图、平台文案或已有视频的适配版本，但不成为新的知识源。Private Pages 按项目、主题、人物和时间自动归入 Book，用户保留确认、改名、移动、合并和拆分权。

Model

四个用户可理解的核心对象

Private Page最完整、默认私密的个人 Context
Shared Page用户确认后的单一 HTML 快照
Distribution针对渠道的内容与媒体适配
Book组织相关 Private Pages 的主题叙事
Library多本书构成可持续使用的个人记忆

知识只在一处保持完整；对外内容都从它产生，并服从用户确认与版本隔离。

Object 01
CURRENT · SOURCE

Private Page · 私密书页

Private Page 是个人记忆与知识源，责任是忠实保存并组织 Context，而不是提前把一切改写成对外内容。

CAPTURE

原始输入

语音及转录、文字、图片、已有视频和用户补充的文件。

CONTEXT

长期背景

相关项目、人物、历史决策、前序书页与当前阶段。

REASONING

来源与判断

可追溯事实、明确标记的推断、待确认表达和行动线索。

PRIVATE

敏感内容

可以保存不适合外传的信息；默认仅用户本人可见。

  • 生成分享页不等于公开，连接社媒账号不等于允许发布。
  • 撤回分享不会删除 Private Page。
  • 平台编辑不会反向覆盖 Private Page。
Object 02
CURRENT · PROJECTION

Shared Page · 分享书页

用户选择展示内容，并确认敏感信息与表达后，Memova 才生成一个 HTML 书页。

四种可见性

  • 仅自己。
  • 已获授权的 Memova 好友。
  • 持有密码的外部收件人。
  • 用户明确设置的完全公开页面。

好友与密码可以同时开启；满足其中一种授权条件即可进入。私域页面默认不被搜索引擎收录。

一个分享快照

  • 每张 Private Page 在 MVP 中最多对应一个 Shared Page。
  • Private Page 更新后，Shared Page 保持原状。
  • 用户点击更新并再次确认后，外部内容才变化。
  • 修改平台文案不会改变 Shared Page。

Explicitly later 团队版、投资人版、家人版等多受众内容分支不属于 MVP。

Object 03
CURRENT · ADAPTERS

Distribution Assets · 发行素材

发行素材从已确认的分享内容派生。平台格式是呈现层决策，不改变长期记忆中的事实与结构。

WEB

HTML

单文件或稳定链接，适合直接阅读、私域分享与打印。

IMAGE

长图与多图

一张长图、多张知识卡或图文页，成为主要视觉输出。

SOCIAL

平台内容

LinkedIn 多图与文案、TikTok 图文轮播、X 单帖或 Thread。

VIDEO

已有媒体适配

剪裁、比例、字幕、封面、配文，或用图片与简单转场制作图文视频。

MVP 不直接生成全新的动态画面或新视频帧。平台标题、字符限制、标签、裁切和比例不会反向污染 Private Page。
Object 04
CURRENT · MEMORY

Book · 持续演化的主题叙事

Memova 根据项目、主题、人物和时间，建议把 Private Page 归入一本书；用户保留最终控制权。

AUTO

自动建议

从 Context 和页面关系中提出归书位置，减少用户维护文件夹的负担。

HUMAN

用户决定

接受、改名、移动、合并或拆分；自动化不剥夺最终控制。

RELATION

保留关系

页面之间保留先后、来源、人物、项目和决策关系，而非只存放在同一目录。

REUSE

成为新 Context

新页面可以调用此前页面提供的项目历史，让长期记忆持续增值。

Explicitly later 从部分 Shared Pages 策展成一册可分享的书，是紧随其后的能力，不是当前 MVP 必须完成的对象。

Lifecycle

权威与更新关系

对象主要用途默认可见性知识源更新方式
Private Page完整 Context 与记忆仅自己是在私人空间持续完善
Shared Page给指定读者的 HTML 快照私域；可明确公开否用户确认后创建或更新
Distribution Assets面向渠道的内容包发布前仅自己否逐平台预览和确认
Book组织相关 Private Pages默认私密组织层自动建议、用户调整
Dogfooding

为什么 HTML 是投影

HTML 适合人类阅读、私域分享、打印和跨设备打开，但它不应成为唯一知识源。

  • HTML 可能为某个受众删减敏感信息。
  • 页面样式与平台格式会改变。
  • 同一知识源需要供 Agent 结构化读取。
  • 分享快照必须与持续演化的私人记忆解耦。
本产品书采用相同模型：Markdown 保存 canonical content，HTML 是可重新生成的人类阅读书页。
