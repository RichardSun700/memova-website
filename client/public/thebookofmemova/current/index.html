从完整 Context
到一张 Page

Memova 把创业者每天产生的语音、会议、想法与项目进展，变成愿意保存或分享的书页，并让这些页面逐渐形成个人知识之书。

Chapter 02 · Page 01Highest authority

跨平台发行是能力；个人出版社与私人图书馆才是产品。

30 Seconds

Agent Quick Read

Memova 正在进行一次小幅但关键的产品扩展：从“自然输入 → 结构化记忆 → 行动”延伸为“完整 Context → 私密书页 → 选择分享或发行 → 自动归书 → 个人图书馆”。第一位用户是创业者。用户以约三分钟语音或文字记录会后复盘、产品决策、客户观察或创始人观点，Memova 结合已有项目、人物与历史 Context 生成默认私密的书页。用户只保存也能获得完整价值；分享 HTML 与社交平台素材必须经过确认。长期价值来自一页页自动形成相互关联的书，而不是社媒排程本身。

Definition
CURRENT

创业者的个人出版社

Memova 把每天产生的语音、会议、想法和项目进展，变成可以保存或分享的书页，并逐渐积累成一本本个人知识之书。

核心对象是 Page，不是 Post，也不是文件夹。Page 首先属于用户的私人记忆；分享和发行只是它经过确认后的可选投影。

用户第一次从约三分钟语音得到一张愿意保存、或愿意分享给某个人的书页，就是 MVP 的第一次价值时刻。
第一把尖刀

扩展过去，而不是推翻过去

FOUNDATION 原有的“自然输入 → 结构化记忆 → 行动”继续成立；CURRENT 新增的是更强的使用、分享与积累闭环。

完整 Context语音、会议、项目、人物和历史决定
Private Page默认私密的完整扩展笔记
Optional Share确认后的 HTML 或平台发行素材
Book按项目、主题、人物和时间建议归书
Library为下一张 Page 提供更好的 Context

用户不需要为了维护知识库而持续整理文件夹。继续工作、说话、学习和思考，本身就是图书馆的输入。这个闭环同时创造立即价值与复利价值：本次输入成为可用成果，历史成果又帮助 Memova 更准确地理解下一次输入。

First User
CURRENT

先服务高 Context、低整理时间的创业者

第一位用户每天产生大量高价值 Context，却没有时间把它整理为可保存、可分享的成果。

01

会后复盘

刚结束的会议、关键判断、承诺与下一步。

02

产品决策

一次取舍、迭代理由和与历史决定的关系。

03

客户与行业观察

尚未整理但值得保留的模式、证据与观点。

04

创始人表达

项目进展、关键转折和准备分享给特定读者的观点。

学生和 SMB 老板属于同一长期用户族群，但不是 MVP 的首屏叙事。学生可把课堂、读书和项目形成学习书页；SMB 老板可把产品知识、客户故事与经营经验形成内容和案例。

Confirmed

已确认的产品边界

INPUT

语音与文字为主

图片可以作为素材；已有视频可以输入。Memova 使用已有项目 Context 扩展本次输入，而不是只做孤立转录摘要。

OUTPUT

HTML、长图与多图

可生成文案、图片顺序、字幕、封面和简单图文视频；MVP 不生成全新的动态画面或视频帧。

SHARING

先私密，再由人确认

用户不发布公域也能获得价值。Shared Page 与每个平台内容在对外出现前都必须经过用户确认。

MEMORY

自动建议归书

按项目、主题、人物和时间归书；用户可以确认、改名或移动，后续 Page 可以复用此前 Page 与项目历史 Context。

  • 直接发布优先级是 LinkedIn → TikTok → X；具体落地服从官方审核、权限与成本。
  • 公域发布是可选动作，私域 HTML 分享与只保存都是完整产品路径。
  • 处理已有视频或把图片制作成图文视频，不生成全新动态帧。
Position

不是另一个社媒 Scheduler

现有多平台工具通常优化“素材 → Post → Calendar → Social Platform”。Memova 优化“完整 Context → Private Page → Shared Page → Book → Library”，社交发行只是 Shared Page 的可选出口。

  • 不是把语音逐字稿换一种排版。
  • 不是必须公开发布才能成立的创作者工具。
  • 不是把所有私人 Context 直接暴露给外部读者。
  • 不是要求用户先设计复杂知识分类体系的文件管理器。
