先私密
再决定分享

Memova 先生成完整的 Private Page。用户可以停在这里，也可以确认一个分享快照、设置访问方式，再选择是否发行到具体平台。

Chapter 02 · Page 03End-to-end MVP

HTML 不依赖社交平台成立；平台失败也不能破坏用户的书页与记忆。

30 Seconds

Agent Quick Read

用户以约三分钟语音或文字开始，并可补充图片、已有视频与项目资料。Memova 结合历史 Context 识别项目、人物、来源、敏感信息和需要确认的表达，先生成默认私密的完整书页。用户可以到此为止，也可以确认一个 Shared Page，并设置 Memova 好友、密码、失效时间或明确公开。用户还可为 LinkedIn、TikTok、X 分别预览并确认平台素材；直发不可用时降级为 Assisted Publish 或 Export Only，不阻塞 HTML。Private Page 自动获得归书建议。发行事实始终保存，基础互动指标只在官方权限允许时同步。

Experience

从一次输入到一本书

分享不是强制漏斗。用户可以只保存、私域分享 HTML、明确公开 HTML，或生成社媒发行素材。

STEP 01

Capture

录制约三分钟语音或输入文字，也可补充图片、已有视频与项目资料。

STEP 02

Understand

关联项目、人物、前序书页和历史决策；识别来源、推断、敏感信息与待确认表达。

STEP 03

Generate the Private Page

生成完整扩展笔记，用户可以编辑、补充或只保存到私人图书馆。到这里已完成核心价值。

STEP 04

Choose a Destination

选择只保存、私域分享 HTML、公域分享 HTML，或生成社媒发行素材。

STEP 05

Review the Shared Page

查看最终外部内容以及姓名、客户信息、内部数字、未公开计划、推断与缺少来源的强陈述，确认后才创建或更新快照。

STEP 06

Set Access

授权 Memova 好友、设置密码与可选失效时间，或明确公开；访问可以撤回。

STEP 07

Optional Social Distribution

分别预览 LinkedIn、TikTok、X 的账号、文案、素材顺序、可见范围和声明，再逐个平台确认。

STEP 08

Auto-book

Private Page 获得归书建议，用户接受、改名或移动；长期由相互关联的 Page 形成个人图书馆。

STEP 09

Optional Feedback

始终保存发行事实；权限允许时异步补充指标，权限不可用时保留未知而不是显示错误零值。

Privacy
CURRENT

Human-led by default

PRIVATE

默认私密

Private Page 创建后仅用户本人可见；连接账号、生成素材或生成 Shared Page 都不会自动公开。

REVIEW

对外前确认

分享快照和每个平台分别展示最终内容、账号、可见范围与必要声明。

VERSION

版本隔离

Private Page 更新不会静默改变外部页面；用户再次确认后才更新 Shared Page。

REVOCATION

可以撤回

撤回链接、修改密码或移除好友权限，不删除私人记忆。

Minimal version rule

  • 每张 Private Page 最多一个 Shared Page 快照。
  • 修改某个平台文案不会改变 Shared Page 或其他平台版本。
  • 同一 Page 的团队版、投资人版、家人版等多受众分支不属于 MVP。

Access rule

  • 好友访问每次检查当前 Memova 权限。
  • 密码不以明文保存，并可修改、撤销或设置失效时间。
  • 好友与密码可以同时开启；满足任一授权条件即可进入。
  • 私域页面默认 noindex, nofollow，媒体地址不能绕过页面权限。

本内部产品书只演示这些规则的结构与叙事，不声称本地静态网站已实现真实登录、密码或生产访问控制。

Distribution

直发是优先级，不是依赖

当前直接发布优先级是 LinkedIn → TikTok → X。具体实现必须服从当时的官方文档、应用审核、账户权限与成本。

状态成立条件用户得到什么
Direct Publish官方接口可用、应用已获批准、用户已授权在 Memova 预览并确认后直接发布
Assisted Publish直发暂不可用，但可以准备完整素材打开目标平台，由用户完成最终确认
Export Only平台无可用发布路径下载或复制平台适配包
01 · LINKEDIN

知识型内容优先

与创业者的产品进展、行业观点和专业表达最接近。

02 · TIKTOK

图文与已有视频

适配图片轮播和已有视频；直发取决于官方审核与权限。

03 · X

短观点与 Thread

适配简短观点和连续表达；接口成本属于实现约束。

FALLBACK

HTML 永远独立

任一平台失败都不影响 Shared Page、Private Page 或其他渠道。

平台审核、权限与成本不能阻塞核心 HTML 书页体验。
Media Boundary

重组已有媒体，不生成新动态帧

MVP 可以处理用户上传的已有视频，也可以把图片、文字、可选配音与简单转场组合成图文视频。

  • 支持已有视频剪裁、比例适配、字幕、封面与配文。
  • 支持图片序列生成图文视频。
  • MVP 不生成全新的动态画面或新视频帧。
Feedback

先保存发行事实，再补充平台指标

A · GUARANTEED

发行记录

平台、账号、发布状态、发布时间、Post ID、公开链接、错误信息与重试状态始终保存。

B · PERMISSION

指标补充

仅在官方接口、应用权限与用户授权允许时，同步平台实际返回的基础互动指标与最近同步时间。

  • 权限不可用时自动退回 A，不阻塞分享或发布。
  • 不显示虚假的零数据，也不把“未知”解释成“没有互动”。
  • 不通过非官方抓取绕过平台权限。
MVP Scope

当前必须成立什么

CAPTURE → PAGE

生成并确认 Private Page

创业者语音/文字输入，结合项目 Context，支持编辑、敏感信息提示和最终确认。

PAGE → SHARE

一个受控分享快照

单文件或稳定链接 HTML，支持好友和密码访问、撤回与可选失效时间。

PAGE → MEDIA

长图、多图与平台预览

LinkedIn、TikTok、X 适配预览；至少一个平台完成真实直发，其他平台可降级。

PAGE → BOOK

归书与发行记录

自动建议归书，用户可确认或移动；保存发行事实，权限允许时同步指标。

Explicitly later · 不是当前承诺

  • 同一 Private Page 的多受众版本，以及复杂团队审批和权限矩阵。
  • 完整社媒日历、批量排程和九个平台全部 Direct Publish。
  • 评论正文回流、总结、自动回复和自动社交互动。
  • AI 生成全新动态视频画面。
  • 以分析仪表盘为主的首页。
  • 从多张 Shared Pages 策展公开书籍的完整体验。
Success

验证 Page 是否真的有价值

ACTIVATION

愿意保存或分享

用户完成一次约三分钟输入，得到一张愿意保存或分享的书页，并完成私域分享或至少一次平台发布。

QUALITY

隐私边界清楚

没有未经确认的敏感信息；用户知道什么私密、什么正在分享；移动端内容不被关键裁切。

RETENTION

第二张 Page 更聪明

用户在一周内生成第二张书页，新 Page 能复用前一页和项目历史 Context，并开始形成一本书。

DISTRIBUTION

愿意交给真实读者

用户愿意把 HTML 发给同事、朋友、家人或投资人，或发布到 LinkedIn、TikTok、X。

Dogfooding

这本书就是一次体验验证

Memova 自己成为第一个被整理的用户：公司 Context 变成 canonical Markdown；主题成为带 ID、状态、来源和关系的 Page；Page 组成三本 Book；HTML 成为团队可读的投影。

团队评审的不只是产品说明，也是在验证 Page → Book → Library 能否让人和 Agent 更快理解一个复杂主体。
