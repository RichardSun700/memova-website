2 分钟知道黄仁勋这次真正想说什么

这不是一场普通芯片发布会，而是在重新定义 AI 时代的会计科目：以前企业买算力是 capex，现在企业买的是 token 产能、agent 劳动力和智能产出。

Based on official replay + manual captions
1

AI 工厂：数据中心不再是成本中心，而是生产智能的工厂

黄仁勋把 GPU 集群、网络、存储、冷却、软件和模型编排合并成一个新概念：AI factory。它的产出不是网页点击，也不是服务器利用率，而是 token、reasoning、agent action。

2

Token 成为新的价值锚：替代一部分劳动力预算

你的笔记里“token 取代劳动力”非常关键。Jensen 的逻辑是，企业会用 token 预算衡量 AI 员工/工程师的杠杆率：不用 AI token 的知识工作者，像在用纸笔设计芯片。

3

CUDA 飞轮：NVIDIA 卖的不是芯片，而是可迁移的整套生产系统

CUDA、CUDA-X、库、开发者、模型优化、网络和 rack-scale 系统形成复利。越多应用跑在 CUDA 上，下一代硬件越容易被采用；越多硬件部署，软件优化越有价值。

4

100 万倍增长：给市场一个“还早”的尺度

他用长期效率增长强调 AI 不是短期硬件周期，而是持续十年以上的 computing transition。重点不是某一代 GPU，而是每一代都把单位 token 成本压低，把可做的任务扩大。

5

35x / 50x 效率语言：市场开始用 token per watt 定价

传统云看 utilization，AI factory 看每瓦、每美元、每秒能产出多少 token。对资本市场来说，这会把估值从“买了多少 GPU”转向“这些 GPU 能生产多少可售智能”。

如果 AI factory 的产出是 token 和 action，那么 Memova 的位置不是“总结笔记”，而是把人类真实世界输入变成 agent 可用的上下文、记忆和可确认行动。

模拟解释：当推理效率提升，Memova 可以把更多长上下文、更多 agent action 和更多本地知识库检索纳入同一个用户预算。真正的产品壁垒会从“谁接了更便宜模型”转向“谁拥有更干净、可控、持续更新的人类上下文”。

1. Natural input

手写、会议、语音、照片、quick note。

2. Memory graph

人物、项目、承诺、偏好、决策、时间线。

3. Agent context

把私有知识转成任意 agent 可读上下文。

4. Token/action

生成日程、邮件、提醒、研究、代码任务。

5. Human confirm

高影响动作不静默执行，用户确认后落地。

产品形态 1：个人 Token Budget 面板

让用户看到每天的 AI token 不是“花费”，而是把笔记转化为行动产能。Memova 可以显示每条记忆带来的 action、节省时间和未来复用次数。

产品形态 2：Local AI Factory

本地 vault 是原料仓，agent 是生产线，Memova 是调度系统。优先做导出、权限、可追溯、可删除，而不是锁定在单一云端模型。

产品形态 3：Five-action Planner

不要做复杂 dashboard。每天把所有会议和笔记压成 5 个最有价值的 action，形成“token -> action -> confirmation”的闭环。

产品形态 4：Memory API for Agents

如果 CUDA 是 GPU 的软件飞轮，Memova 的飞轮应该是个人上下文 API：OpenClaw、Codex、Claude Code 等 agent 都能读，但用户掌控边界。
